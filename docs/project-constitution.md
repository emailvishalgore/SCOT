# Project Constitution

Project Name:
SCOT Community Operations Platform

Development Method:
Spec Driven Development

Rules:

1. Never generate code before reviewing specs.
2. Specs are source of truth.
3. Ask for clarification when requirements conflict.
4. Design for modular architecture.
5. Finance module must remain plug-and-play.
6. Preserve season-based architecture.
7. Do not remove future extensibility.
8. All changes must update relevant specs.
